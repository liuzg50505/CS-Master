CS5002 Programming for AI
Lecture 8: Validation
Dr Paul Henderson
University of Glasgow
Hyperparameters
● k is a "hyperparameter"
● hyperparameters…
○ are parameters that change the algorithm or model in some way
○ are not learned from data
○ affect ﬁnal performance
● want to choose the best values
○ …but how?
Perfect training?
● "we achieved zero error (e.g. MSE) on training data"
○ i.e. our gradient descent found a 'good' solution
● does this mean we succeeded?
○ is it the best possible model?
Key term
● "overﬁtting" = ﬁtting the noise as well as the data
○ training error doesn't tell us this
● tends to happen for large-capacity models
○ high k polynomial
○ big neural network
● …and small datasets
○ 10–100 rather 10K data
Validation
● training error does not tell us if we overﬁtted
○ zero training error ≠ good model
● good models must generalise
○ …i.e. work on new data
● k with best training error is not best k
Validation
● we must always evaluate our model on separate data
○ …i.e. not the data it was trained on
Validation
● we must always evaluate our model on separate data
○ …i.e. not the data it was trained on
● call this "validation data"
● different to the test data
○ only look at test data once (ideally)
○ cheating to use it for anything
● different strategies to select it
Validation sets
● simplest: reserve a ﬁxed chunk of the dataset for validation
● the "validation set"
…or "validation split"
Validation in sklearn
● in scikit-learn: train_test_split
○ ﬁne for validation set too!
● can call it twice
○ once to get train+val separate from test
○ once to get train separate from validation

Validation sets
● to choose a hyperparameter like k…
○ split into training & validation sets
○ for each k…
■ train model on training set
■ predict on validation set
■ get mean error (on validation set)
○ choose the k with lowest mean error
Validation sets
● what if we have too little data?
○ separate validation set  ⇒  even less training data
○ small validation set  ⇒  might be unlucky
○ validation data may be unusually hard / easy / strange
Cross validation
● solution: lots of validation sets
● don't just choose once, choose many times
○ repeat experiment for each
○ average over them
Cross validation
● to choose a hyperparameter like k…
○ split into M "slices"
○ for each k…
■ for m = 1 … M:
● take mth slice as validation set
● take other M–1 slices as training set
● train model on current training set
● predict on mth validation set
● get mean error (on mth validation set)
■ average over the M slices
○ choose the k with lowest mean error

cross_validate
from sklearn.model_selection import cross_validate
model = LinearRegression()
cv_results = cross_validate(
model,
X_train_and_val,
y_train_and_val,
cv=6,
scoring='neg_mean_squared_error'
)
print(cv_results['test_score'])
