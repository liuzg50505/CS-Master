Problem description
Numbers in electronic devices are usually displayed  via seven-segment panels. Each
number consists of 7 strokes, labeled as a ~ g as shown below.  The strokes are
internally connected to one-bit memory cells each storing a 0 or 1 (indicating off/on).
In this assignment, we need to work with command se quences which control the
strokes a ~ g to display desired numbers.
Assume we have one seven-segment panel here and we want to display a sequence of
numbers at different time. When the hardware device  starts up, all memory cells
connecting a ~ g stores 0 by default (i.e. all strokes are off  by default).
Suppose we want to display number 1. From the diagram, we need to turn on stokes b
and c. We therefore issue the commands b+c+ . The first alphabet ( b) indicate which
stroke, and the symbol after it is + or -, with + indicating “turn on” and – indicating
“turn off”. Now the strokes are ready, we then disp lay the number using the
command  = . Therefore the command sequence is
b+c+= .
In this assignment, we have the following assumptions on the command sequences:
 The alphabet can only be a/b/c/d/e/f/g exist always in lowercase.
 The strokes must be mentioned in order from a to g. (so c+b+=  is wrong) .
 No spaces in the command sequence.
 The sequence must not be redundant. (e.g. to display “1” when all strokes are
currently off ,  b+ b+ c+=  and a-b+c+=  are wrong (as a is already off) ).
Now suppose that after displaying 1, we need to display 4. From the diagram, 4
consists of b, c, f and g. Since b and c are already on (as we just displayed 1 – the
on /off  commands issued earlier persist) , we only need to turn on f and g. Therefore
the command for displaying 4 now is
f+g+= .
Now suppose we finally want to display 7 after 4. From the diagram, we need to turn
on a and turn off f and g. Therefore the command is a+f-g-=.
To summarize, the command sequence for displaying 147  is b+c+=f+g+=a+f-g-=.
In this assignment, the program needs to  keep printing the prompt “ Command?  ”,
then reads a single capital-letter  command and process the subsequent input
accordingly. All commands are independent from each other
.
Command Q:
 Q means “quit”. The program prints the word “ End ” and terminates.
 All testcases contain only one Q command in the last line.
 Before the Q command, there can be arbitrary number of N or S commands.
Command N:
 N means “number”. The program will read the number s equence we need to
display and generates the command sequence correspondingly.
 We assume all a~g  strokes are off when processing each N command.
 The length of input number sequence is 1..100 digits.
 The command N and the number sequence are separated by a single space. Other
than that there’s no space character within the number sequence.
Command S:
 S means “sequence”. It is the opposite of the N command. The program will read
the command sequence and print the corresponding number sequence.
 The length of the output number sequence is 1..100 digits.
 We assume all a~g  strokes are off when processing each S command.
 It is guarantee that the command sequence is valid (i.e. always result in valid num,
it’s not redundant and the strokes are in order (e.g. a+b+  is never written as b+a+ )).
 The command S and the command sequence are separated by a single  space.
Other than that there’s no space character within the command sequence.
