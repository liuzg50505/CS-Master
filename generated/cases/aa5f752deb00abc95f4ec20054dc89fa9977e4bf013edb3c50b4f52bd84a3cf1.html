School Management System
Introduction
This document describes the development of a web-based school management system
that has replaced outdated paper records at St Alphonsus Primary School with a secure
and easy-to-use system for managing student, teacher, parent and class information
(Connolly & Begg, 2015). The goal of this project was to create an accessible system for
teachers who are not technically minded, which included the design of a database, the
implementation of a web interface using PHP and HTML, and the configuration of the
web server.
Entity Relation Diagram
An Entity Relation (ER) Diagram was created to show all the components of the system
and was designed in accordance with the database design best practices recommended
by Connolly and Begg (2015). The main entity types identified in the system were; Pupil,
Teacher, Parent, Class, and Pupil_Parent for managing the pupil - parent relationships
and to make sure the design of the database is as simple and efficient as possible and
the ability to enforce the relationships between the entities.
Technologies Used
PHP
PHP was selected to serve as the server-side scripting language because PHP can
operate on most all web servers and support MySQL database, (PHP.net, 2023). Blum
(2018) suggests that PHP has an extensive range of documentation to support users in
getting started easily because PHP is easy to read and understand, making it the most
appropriate server-side language for developing web applications. PHP will perform a
variety of tasks for the web-based School Management System, such as: User
Authentication, Form Processing and Managing Database Interactions.
MySQL
MySQL is being used by St Alphonsus Primary School as the DBMS for their School
Management System. Some of the reasons Altistics selected MySQL were due to its
Reliability, Performance, and its ability to work with PHP (MySQL, 2023). According to
Connolly & Begg (2015), the ability of MySQL to apply foreign key constraints to enforce
referential integrity makes the database easier to work with in terms of simple user
queries, as well as in relation to the complex queries that are required by administrators
when generating reports.
HTML & CSS
The front-end of the school management system will use a very lightweight, user-friendly
interface built entirely with basic HTML and inline CSS styling. The use of semantic
HTML creates a logical and readable structure for the front-end page and provides an

Apache
The Apache Web Server was selected as our hosting platform due to its capabilities
surrounding mod_php, and its ability to handle many simultaneous users (W3Schools,
2023). According to W3Schools (2023), this makes it an effective hosting solution for
PHP applications.
Other Considerations
Prior to finalizing the technology stack, alternative technologies were considered
(Connolly & Begg 2015). Oracle Database was eliminated due to high costs and
complexity; MariaDB, which is a fork of MySQL, was not chosen because MySQL is
more widely used and supported. Lastly, MongoDB was rejected because it was
developed as a NoSQL database with the goal of providing scalability and flexibility but
not data consistency, which is essential in education-related records (Connolly & Begg
2015).
Development Process
The school management system was developed in phases in a structured and iterative
manner, allowing for the complete testing of each component before beginning work on
the next.
In the beginning stages of the development of the school management system, the
design of the application database was created to satisfy the application's requirements.
The application's six primary domain objects are: Pupil, Teacher, Parent, Class, the
junction table of Pupil and Parent, and User Accounts. A relationship map was created
showing how these primary objects relate to each other.Once definitions for the entity
relationships had been established, a MySQL Schema was created with foreign key
relationships to provide data integrity throughout all of the tables. Sample test data was
added to the database to support testing of core functionality such as sample student
records, teacher assignments, and configurations for classes. This also provided a
foundation to test all other areas of the application on.
After sufficient testing of the database, work began on building out the core components
of the application itself beginning with the Database Connection Layer and the User
Authentication System. A mysqli connection script was created to provide a secure
database connection including error handling to allow graceful handling of connection
failures. The User Authentication System provides both Registration and Login modules
with front-end HTML5 validation to ensure the completeness and accuracy of the data
entered on the forms. The Registration Form contains the necessary information required
to register an account, including Username, Password and Email. The Login Form allows
a user to validate their User Credentials against the records saved in the Database.
The Login Process also includes Basic Role-Based Access Control (RBAC) to
differentiate between Administrative Users (who manage the system settings and user
accounts) and Regular Staff Users (who manage daily student and class management
processes), allowing users to only access functionality appropriate to their level of
responsibility.
Following the creation of the User Authentication and Database Components,
development progressed onto Core Management Features (i.e. Pupil Administration
Forms for creating, editing and deleting pupil records; Teacher Administration Functions
for adding new teachers, modifying teacher contact information and assigning teachers
to classes).
Pupil Management (Admin) allowed for multiple pupils to be assigned to multiple
guardians through the junction table Pupil_Parent. Class Management Tools allowed
administrators to determine how many pupils could be assigned to each class by
creating a maximum pupil count per class. A validation function prevented a teacher from
having more pupils assigned than established maximum quotas.
Upon clicking on a teacher's class, Staff can view the total number of pupils currently
enrolled in the class as well as the number of spaces currently available for enrollment.
The validation process checks for available spaces in real time.
Conclusion
The school's web-based management system has successfully and securely replaced St
Alphonsus Primary School's previous paper records. The HTML, PHP, MySQL/Apache
stack provides the required balance of performance and ease of development which
aligns with the requirements of the school.
