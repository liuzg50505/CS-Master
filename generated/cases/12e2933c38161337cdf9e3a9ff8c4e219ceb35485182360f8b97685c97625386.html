Project 1
ECS 32C Data Structures and Algorithms in C
1 Problem Statement
For this assignment, you are going to implement a comparison-based sorting algorithm that simulta-
neously makes the optimal number of comparisons (O(nlogn)) and requires minimal additional space
(O(1)). Such a sorting algorithm is also referred to as an “in-place” sorting algorithm in literature.
Here, the additional space of a sorting algorithm is defined as the memory it requiresin additionto
the input array. To make sure that your sorting algorithm is (a) general and (b) comparison-based,
the array that you are required to sort will be acustom data structure. The way your program should
access and modify this data structure is described in Section 2.
What you are given.On Canvas, go to “Assignments”→“Project-1” and then download the zip
file “Project1.zip”. This zip file contains all the files that are needed for this assignment. It will contain
a folder, and within it, there will be several subfolders that describe the project’s structure. Although
this assignment is simple, the structure of the project also serves another purpose: It shows you how
large C programs should be structured and how you will find C programs in the wild.
What you should do.You are required to complete the “IndexSorter.c” file. Specifically, you
are required to complete theSortByIndicesfunction. This function should implement your sorting
algorithm. To understand what the arguments represent and exactly what the function should do,
read Section 2.
How you can test.Once you have completed the sorting function, run the provided script to test
your program against the inputs provided. The script will first compile your file and report any issues
in your program. More details are provided in Section 3.
How to submit.Your submission will be on Gradescope. You should submitONL Ythe “Index-
Sorter.c” file to Gradescope. A copy of the project is present in the internal grader of Gradescope,
and we will simply replace the default “IndexSorter.c” file with your submission and run a copy of the
checker provided to you on Gradescope’s servers. For more information see Section 4.
Grading.If you implement the intended sorting algorithm correctly, then you should receive 100
pts. Otherwise, you may receive partial credit. Your program will be run against 10 set of test cases.
Passing all the tests in a single set will get you 10 points.
2 Data Structures and Project Structure
Inside the project folder in the “Project-1.zip” file, you should see the files and folders listed below.
You do not need to understand their purpose, but if you are curious about why the project is orga-
nized in this way, you may read the descriptions below. You should only modify the contents of the
“IndexSorter.c” file within the “src” folder.
2.1 Project Structure.
i (bin). This folder contains all the “binaries” for this project. A “binary file” is intended to only
be read by machines and describes your code in low-level language for the operating system to
execute. This will be empty when given to you. Once you run scripts to test your program,
1
binaries will appear in this folder, created by the script. If you wish to reset the project, run the
command “make clean”, from the project folder in the terminal.
ii (include). This folder contains all the header files that are necessary for your project. For this
assignment, we have created a custom data structure designed for use exclusively in this project.
All such header files will be placed inside this folder. In general it is good practice to use header
files in a large project.
iii (test data). This folder contains sample input and output files that you can use to test your
program. The provided script “run.sh” automatically runs a checker program and tests against
all the input files provided.
iv (obj). This folder contains all the necessary objects to link and build an executable for your
program. Initially, this folder will be empty. As you build your projects, this folder will con-
tain machine-translated code for parts of your program that can be reused to speed up later
compilations.
v (src). This folder contains all of the source files for your project. These are the main program
files of your project.
vi (Makefile). A “make file” for a C/C++ program is a compact file that generates and executes
custom compilation commands. You provide all the options for the compilation command that
you need to generate an executable in your make file. For e.g., the libraries to be linked, packages
to be installed, code to be run, etc. You do NOT need to know how make files work, but you
are encouraged to read up on how to use them.
For this project, the make files are used in the “run.sh” file, so you need not know what it does.
If you wish to use it, note that it is designed to compile the “main.c” program into “./bin/proj1”.
If you run the command “make all”, this will execute the compilation command provided for this
project. You will notice that the (i) “DataCollection.c”, (ii) “IndexSorter.c” and (iii) “main.c”
files are compiled into objects “DataCollection.o”, “IndexSorter.o” and “main.o” respectively.
Finally, to generate the final executable, the three objects are linked together into a single
executable “./bin/proj1”.
vii (“./src/main.c”). The “main.c” C program reads (i) a set of records and (ii) how to sort them
from the standard input stream and then invokes your “SortByIndices” function to sort it and
print the list of records to the standard output stream.
viii (“./src/DataCollection.c”). This C program contains the data structure that stores the list of
records of people’s details. Informally, it creates an integer arrayAof size 4·nwherenis the
number of people. Thei th person’s details will be stored inA[4·i], A[4·i+1], A[4·i+2], A[4·i+3].
You are provided a function “QueryRecord(i, e)” that takes an integeri(0≤i < n) and an element
e(0≤e <4) and returnsA[4·i+e].
Important:You should not change the contents of this file.
Input array .The input arrayP[0], P[1], . . . P[n−1] that you are required to sort represents a record
of personal details of a set of people. Each person has a (i) unique ID, (ii) DOB - Date of birth as
an integer in yyyymmdd format, e.g., integer 20251105 represents “5th November 2025”, (iii) Height

cannot explicitly access the contents of the input array. However, through functions provided in the
“./src/DataCollection.c” file, you can read and manipulate the input array.

2.3 Your algorithm
You should complete the functionSortByIndicesin the “./src/IndexSorter.c” file. The descriptions
of the function arguments and their intended functions are provided below. First, we describe the
procedure for comparing two sets of records which is described by an additional arrayorderthat will
be given as an additional input.
Comparing two records.You are given an arrayorder[4]. The elements ofordercan be positive
or negative. The absolute values oforderare distinct and denote the priority ordering of the elements
when comparing two people. Iforder[i] is negative, then you should sort elementiindescending order 1,
otherwise you should sort them inascending 2 order. For e.g. iforder={−1,2,−3,4}– recall that here
1 denotes ID, 2 denotes DOB, 3 denotes height and 4 denotes weight – then ID should be prioritized
first and then DOB, and then height and lastly the weight. Ordering within ID, height should be done
in descending order since they are negative and the rest in ascending order since they are positive.
More specifically for the givenorderarray you should sort the records such that: (i) people with
higher IDs appear first, (ii) if two people have the same ID, then the person bornearliermust appear
first, (iii) if two people have the same ID and same DOB, then the person withlargerheight appears
first, (iv) if two people have the same ID, DOB and height, then the one withsmallerweight appears
first and finally (v) if two people have the same set of records, then they may appear in any order.
The function to complete:SortByIndices.You should complete the function
SortByIndices(indices[], n,order[], m).nandmare the sizes of the arraysindices,orderrespectively. It
is guaranteed thatm= 4 and that all elements oforderare distinct with absolute values in{1,2,3,4}.

3 Local Testing
Run the script “run local tests.sh” to run a script that will automatically compile and generate a
binary file of your program.
The script will also run and test your program against the inputs provided and relay to you
whether they produced the same output. Note that the checkerdoes notcheck the running time of
your program.
4 Instructions for Submission
SubmitONL Ythe “IndexSorter.c” file on Gradescope. A copy of the project is present on Gradescope
servers and the file that you will upload will be copied and a similar script will run and test your sorting
function for correctness, time complexity and space complexity.
The time limit provided for your program to sort is 2 seconds. The space limit given to your
function is 1000 bytes. This means that if you declare more than 250 integers, then your program will
have failed.
5 Grading
Your program will be tested against 10 test files. Each test file contains multiple test cases. Passing
all the test cases within a single test file will get you 10 points. In each test file, if your program takes
more than 2 seconds, or your sorting function takes more than 1000 bytes of memory, then you will
receive 0 points for that file.
Your program will also receive an error if you use the default provided “qsort” function in the
“stdlib.h” library. You are expected to implement the sorting algorithm from scratch.
