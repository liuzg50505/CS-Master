Role Based Login
Task 1
• Make a login for your school system
• Consider what roles are needed and what data each should
have access to (i.e. a teacher should not be able to see details
of other teachers and a student should only see their own data)
• Create different landing pages for the different roles
• Make considerations on how the role is determined at login and
how the login table interacts with the rest of your database
Task 2
• Ensure you have finished your data retrieval forms
• Making use of the php and sql covered this week, create the
data retrieval pipeline (i.e. user uses form to determine what
data they want, php processes the request and send the
appropriate select query to the database, the database
produces a result which you process in php and show to the
user using html tables)
• Consider the use of conditions in the WHERE clause to limit the
data shown
• Make use of dropdown wherever possible
Task 3
• Following from the previous task, retrieve data from multiple
tables
• Make use of joins to get connected record from multiple related
tables (i.e. when showing the details of a student also show
which parent/guardian they have)
• Try to ease any data duplication that may occur from this
