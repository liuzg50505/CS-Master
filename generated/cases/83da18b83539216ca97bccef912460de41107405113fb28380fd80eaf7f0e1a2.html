Block 2: Back End
Development
Dynamic Learning Day 3
Objectives
Consider what useful data users want to see
Create SQL queries to make use of in final project
Enhance Front-End elements
Task 1
• If you haven’t done so already, finish designing and implementing the database
• Keep in mind to go through the normalisation process and make the full ERD
• Implement the database in MySQL and populate your tables with sufficient (at
least 20 per table where possible) records
• Create and run queries that at a minimum:
• Get the full details of a student
• Get the full details of a teacher
• Get all students in a class
• Get the name of a teacher for a class
• Get the details of a parent/guardian based on the name of a student
• Get all students that have a particular parent/guardian
• SAVE these queries to a file as they will very likely be useful when integrating
SQL into PHP
Task 2
• If you haven’t done so already, finish designing and
implementing the front-end
• Make sure you have a good interface (no need to be fancy, but
it should be clear and easy to use)
• Create forms such that a user could insert data into any of your
created tables (except junctions)
• Create forms such that the user can extract at least the data
from the previous activity from your database
Task 3
• The assignment brief gives you the minimum requirement
• You can go beyond in designing your system and should do so
• Think what other data a school might need to gather and keep
track of and create new tables
• Consider security, and create a login form for the interface and
a matching table in the database to store login information
(think how this can fit into your ERD)
• Consider how you will present the data to the user and create
the PHP/HTML elements for this (consider lists or tables)
Extension
• IF you have finished all other work, look into getting data from forms
in PHP (links at the bottom)
• Implement this and start seeing how to extract data into PHP and
send it to an external source (can use text files for now)
• Keep in mind this will be covered in detail on Monday
https://talent500.com/blog/data-collection-with-php-creating-forms-
setting-up-and-validation/
https://www.w3schools.com/php/php_forms.asp
https://ostraining.com/blog/coding/retrieve-html-form-data-with-php/
Keeping in Touch
• Between 3:00pm and 3:30pm send the lecturer a screenshot of
your most recently completed task (screenshots of partial
completion are also fine)
• Repeat the above between 4:00pm and 4:30pm
• The lecturer will be available throughout the whole period (2pm-
6pm) to answer questions of provide guidance if needed
• Communication is mandatory in order to show engagement
Submitting
• An assignment Dropbox will have been created on Teams
• All students should have gotten a notification, but you can also find it
in the assignments tab in the Back-End Development Team
• The submission should consist of a zip file with all work done today
• NOT ALL TASKS NEED TO BE COMPLETED – A significant
amount of work was given for today as everyone is at different points
in their assignment work. Make sure to complete at least one task.
• The final deadline was set for 5:30 PM
• If you submit the file and engage with check-ins, you will be marked
as having attended today
