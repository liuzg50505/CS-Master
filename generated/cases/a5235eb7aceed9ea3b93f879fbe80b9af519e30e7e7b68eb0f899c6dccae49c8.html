INFO20003 Database Systems 4
Enterprise Data Model (EDM)
© University of Melbourne
• An Enterprise Data Model (EDM) is a comprehensive,
integrated map of all the data used across an entire
organization.
• Serves as the "single source of truth," breaking down
departmental silos to ensure that data, like a "Customer" or
"Product", is defined and structured consistently everywhere.
• Holistic Scope: Unlike a project-specific database, an EDM
covers the entire enterprise, mapping how data from
Finance, Sales, and HR interrelate.

INFO20003 Database Systems 7
Systems Definition
© University of Melbourne
• Foundational phase where the scope and boundaries of the database
system are established.
• Before any tables are created or SQL is written, this stage ensures that
the project team understands what the system is supposed to do and who
it is for.
• Defining the Scope: This involves identifying the major user views (e.g.,
Marketing, Finance, or Customer Support) and determining which
business processes the database will support.
• Establishing Boundaries: It clarifies not just what the system will do, but
also what it won't do, preventing "scope creep" later in the development
process.
• Identifying User Views: Since different departments often need different
"slices" of the same relational data, this stage identifies these unique
requirements early on.
• Interface Identification: It determines how the database will interact with
other systems, such as third-party APIs, web front-ends, or legacy
internal software.
