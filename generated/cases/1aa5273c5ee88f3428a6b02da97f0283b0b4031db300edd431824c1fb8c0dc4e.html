Getting Started with R Markdown
This is a quick guide to useR Markdown on exercises1. Before using this guide, you should be already familiar
with R and RStudio.
Background and Installation
R Markdown is a simple but useful format for authoring HTML, PDF, and MS Word documents and
integrating plain text information and code into one compiled document. For more details about R Markdown
see http://rmarkdown.rstudio.com.
Figure 1: R Markdown window
For usingR Markdown to create PDF files, you need to installtinytex package by opening R and running:
#install.packages("rmarkdown") #Optional: it should be installed with R studio
install.packages("tinytex")

Creating and CompilingR Markdown
To useR Markdown , we recommendRStudio.
Creating an .Rmd file
Creating an .Rmd is similar to creating an.R file. To begin, openRStudio and select File > New File > R
Markdown.
RStudio will ask you to give the document a title and select the default output. Choose PDF for rendering
PDF files.
The default .Rmd includes some sample text to show new users the kinds of information that can be entered.
You can simply remove the sample code except the following:
```{r setup, include=FALSE}
knitr::opts_chunk$set(echo = TRUE)
``` which is necessary for setting up theR Markdown , and start adding your own content.
Compiling a .Rmd file
After editing the content, the last step in preparing your document is to compile the.Rmd; this is done
through a process called knitting. To do this, simply click on theKnit button when you are ready. Remember
that an .Rmd file will be read and evaluated all at once. If there is any issue with the code, it will produce an
error and the document will not be knitted. You can test to knit the example.Rmd created when you create
a new .Rmd file and see the output of a finished PDF document.
Using R Markdown
R Markdown accepts both code and regular text.
Code chunk
Anything you wantR to evaluate (i.e. your code) should be placed inside a code chunk. To make a new chunk,
select a blank line and type out three backticks followed by curly brackets that contain the letter r. Skip a
few lines and type three more backticks to close the chunk. You can also add a chunk by selecting Code >
Insert Chunk or use keyboard shortcut (Win: Ctrl+Alt+I; Mac: Cmd + Opt + I). Any lines inside a code
will appear gray.
The curly brackets include chunk expression which usually looks like this```{r [name], [options]} . The
r means it isR code which can be followed by a custom name defined by you for the chunk. Note that you
don’t need to always name the chunk, but if you do, every chunk should have a unique name otherwise
RStudio will report an error. After the comma, it’s the options for running the chunks, such as whether to
include or evaluate the chunk, whether to produce the output, how the output should look like, etc. For
example, in the setup chunk, theinclude=FALSE means the chunk won’t be included in the final PDF filed.
Other options includeeval, echo, message, warning, etc.

There is another way to add code by inline code. This allows you to write sentences as you would normally,
3
but also integrate the results of you code. This is done by putting your code inside of r instead of a regular
chunk. Here’s an example. Instead of typing ...
• The sum of 12 and 45 is: 57.
You can type ...
• The sum of 12 and 45 is:`r 12+45 `
Which will output ...
• The sum of 12 and 45 is: 57
This can especially come in handy when dealing with methods involving random process, such as Bayesian
MCMC and certain machine learning methods, which won’t produce exactly the same results every time.
You can use inline code in your description and interpretations so you don’t need to change it every time to
re-run the .Rmd file. For the same example, to calculate12 + 45, you can use a code chunk ...
(number <- rnorm(1) + runif(1))
## [1] -0.5439231
Instead of typing ...
• The number is: 2.245712
You can type ...
• The number is:`r number `.
It will give ...
• The number is: -0.5439231.

Regular text and math notations
Outside the code chunks is the regular text. It incorporates bothMarkdown and LATEX expressions. You can
use #, ##, ### for different levels of headings and use * (unordered) and 1. (ordered) for itemizing:
Heading 1
Heading 2
Heading 3
• Unordered item
• Unordered item
1. Ordered item
2. Ordered item
Plain text...
Inline LATEX equations can be written in a pair of dollar signs using the LaTeX syntax, e.g.,$Y = \alpha
+ \beta X$ (actual output:Y =α+βX); math expressions of the display style can be written in a pair of
double dollar signs, e.g.,$$Y = \alpha + \beta X$$ , and the output looks like this:
Y =α+βX
