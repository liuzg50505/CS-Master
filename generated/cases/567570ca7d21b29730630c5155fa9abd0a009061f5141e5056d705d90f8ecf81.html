1 Classification
The regression models discussed so far are appropriate when the outputyis quantitative.
But, in many machine learning tasks, the output is categorical. For example, in email
spam filtering, the output is categorical, indicating whether or not the email is spam
(see further examples provided in Week 1 Lecture Notes). In this case, when the output
is categorical, the learning task is known asclassification.
There are many possible classification techniques that one might use to predict the
class of a new inputx ∗. Typically, this process first involves computing theconditional
probabilitythat the outputy ∗ belongs to each classc∈ {1, . . . , C}:
p(y∗ =c|x ∗,D),

This very simple classifier is called theBayes classifier. In a two-class, or binary

tional probabilities at 0.5, that is, ify∈ {0,1}, we predict class one ifp(y ∗ = 1|x∗,D)>
0.5 and class zero otherwise. The set of input values where the conditional probability
is exactly equal to 0.5 is called theBayes decision boundary; it separates the input
space into the regions determining the predicted class.
In machine learning, classifiers can be built and trained using agenerativeapproach
or adiscriminativeapproach. Generative classifiers create a joint model of the form
p(y,x) =p(y)p(x|y) and then compute the conditional probabilities usingBayes rule:

This approach is termed generative as it specifies how to generate the inputs for each
class. For example,naive Bayesmodels are generative classifiers. Discriminative clas-
sifiers are more popular; they model the conditional probabilityp(y|x) directly. This
approach is called discriminative, since it discriminates between class labels but cannot
generate input examples of each class. This week, we will focus on logistic regression,
which is an example of a discriminative classifier.
