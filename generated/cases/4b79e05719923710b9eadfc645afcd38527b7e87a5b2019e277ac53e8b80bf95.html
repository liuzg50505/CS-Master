Block 2: Back End
Development
Dynamic Learning Day 2
Objectives
Finalise design for school database
Implement database using PHPMyAdmin
Populate tables with sufficient data to showcase relationships
Task
• Over the past week you have been working on designing and building your
database for the assignment
• With knowledge on normalisation and the standards of creating ERDs, finish your
designs and make sure to include:
• All entities and attributes
• Relationships with correct cardinality that explicitly connect primary to secondary keys
• Clear data types
• Considerations for the wider needs of the school
• Once your design is complete, you can translate it 1-to-1 to a database
implementation using PHPMyAdmin
• You can use the GUI or employ SQL for this
• Just like with the ERD make sure to add all attributes, be careful of data types
and create all relationships
• Once the database is created start adding data into it using the INSERT query
Design
• Ensure your ERD has:
• At least 5 entities (there’s at least 1 junction table needed)
• Only valid relationships (only 1-to-1 and 1-to-many)
• Data types for all attributes (including keys)
• Clear labels for the keys
• Relationships that do not simply connect to an entity, but the exact attribute
the relationships is based on
• Make sure to normalise your database before the final version of the
ERD
• Write a few sentences as to how you have normalised your database
and why it meets the 3 normal forms (this is needed for the report)
Implementation
• Create the database and tables for your assignment
• This can be done via GUI or via SQL statements (consult the slides from Monday
and Thursday of week 2 for a refresher on how this can be done)
• Ensure that you implementation meets your design
• Remember the “many” side of a relationship is always on the foreign key table
• Remember to create the tables that contain the primary keys in a relationship first
• Make use of the correct data types
• Ensure all primary keys are set as auto increment (unless your design does not
use ID)
• Include all attributes
• Once you are done, check the designer tab of the database to ensure the
resulting ERD matches the one you have created
Populate
• Once the database is implemented, begin adding data
• Make sure to add data to tables containing the primary keys in
relationships first as to not break referential integrity (use cascade in
general)
• You can use the GUI to add data, but this will be slower
• Try to make use of INSERT INTO instead to add multiple records at once
• You ARE ALLOWED to use AI to generate data to insert in your database
• Make sure all tables are populated (including any junction tables) and
there is enough data to test all your relationships
• Ideally run a few SELECT queries to check that relationships are working
as intended
Keeping in Touch
• Between 3:00pm and 3:30pm send the lecturer a screenshot of
your most recently completed task (screenshots of partial
completion are also fine)
• Repeat the above between 4:00pm and 4:30pm
• The lecturer will be available throughout the whole period (2pm-
6pm) to answer questions of provide guidance if needed
• Communication is mandatory in order to show engagement
Submitting
• An assignment Dropbox will have been created on Teams
• All students should have gotten a notification, but you can also find it
in the assignments tab in the Back-End Development Team
• The submission should include:
• A Word document (.docx) containing your completed ERD (if the diagram is
too big to fit neatly in a word document, submit it as its own image file) and
your explanation of how normalisation was done
• A file containing your database (.sql) generated directly from PHPMyAdmin
(check the next slides for a refresher on how to export)
• The final deadline was set for 5:30 PM
• If you submit the file and engage with check-ins, you will be marked
as having attended today
Exporting a Database
• In PHPMyAdmin, select the database you wish to export
Exporting a Database
• From the top menu choose “EXPORT”
Exporting a Database
• Depending on your version, this might look different, but there are
only two things to consider:
• Make sure the format is SQL
• Don’t touch anything else other than the Export/Go button
Exporting a Database
• Select where to save the resulting .sql file (it will
automatically be given the name of the database)
• If you open it in notepad, you’ll see it just contains a set of
SQL commands in order to recreate and repopulate your
tables
