1. Data Warehousing (DW) (MCQs + Short Answer: Core Concepts &Comparison)
1.1 Basic Definitions & Purpose
Data Warehouse (DW): Centralized repository for structured, historical data; designed for Online
Analytical Processing (OLAP) (analytical tasks, not transactions).
Key Role: Supports data-driven decision-making via complex queries, trend analysis, and business
intelligence (BI).
Why DW for Big Data?: Traditional RDBMS focuses on transaction processing; DW handles
aggregated historical data to complement Big Dataʼs volume/velocity needs.

1.4 ETL Process (Critical for Short Answer/MCQs)

Purpose: Prepare data for DW by standardizing and optimizing it.
3 Stages:
. Extract: Gather data from heterogeneous sources (OLTP DBs, logs, external systems like
suppliers).
. Transform: Clean (remove errors), filter (keep relevant data), and standardize (apply business
rules: e.g., merge "NY" and "New York" into one value; create surrogate keys).
. Load: Store transformed data into DW using optimized structures (e.g., star schemas); activate
constraints (uniqueness, referential integrity) for data quality.

1.8 Modern DW Trends & Challenges
Trends:
Cloud-based DW (Snowflake, BigQuery, Redshift) for scalability.
Lakehouse (hybrid of data lake + DW) for structured/unstructured data.
Real-time analytics integration (Spark Streaming).
AI-driven query optimization.
Challenges:
Data integration complexity (multiple sources).
ETL performance bottlenecks.
High storage/maintenance costs.
Security/compliance risks.
2. NoSQL Databases (MCQs + Short Answer: Types, Comparison, UseCases)
2.1 Basic Definitions & Motivation
NoSQL: "Not Only SQL" – databases for unstructured/semi-structured data; designed for Big
Dataʼs 3Vs (Volume, Velocity, Variety).
Why NoSQL?: Traditional RDBMS limitations (rigid schema, vertical scaling only, expensive joins,
ACID overhead) fail to handle Big Data.
Key Role: Complements (not replaces) DW/RDBMS; supports real-time web apps, IoT, social media.
