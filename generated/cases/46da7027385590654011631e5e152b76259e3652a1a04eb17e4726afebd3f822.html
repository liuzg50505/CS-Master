WINDOW FUNCTION
Define analytic operators over a subset of rows that are linked to the
current row
 Window function keeps the individual identity of each row instead of
merging rows into a single result
 Benefit: analytics-on-the-fly, study data within a specific range in a
flexible way, encourage parallelism.
Define rows subset using: PARTITION BY within OVER
 It divides the rows into partitions that share the same value of the
PARTITION BY expression(s)
 The window function is operated on each partition separately and
recalculated for each partition.

OVER & PARTITION BY
Comparison: Both calculate the TOTAL_REVENUE per department
However,
 GROUP BY collapses all the columns keeping only the grouping attribute
to associate the aggregation results with each department.
 PARTITION BY still shows all the columns adding an extra one
referring to aggregation information per group in parallel.
 PARTITION BY query output has as many rows as the number of the
records in the table.

OVER OPERATOR
OVER defines a window: a set of rows over which a function is applied.
PARTITION BY with OVER restricts the function’s scope to rows in the
stated expressions that have the same value.
Window function per department—keep all the rows in output
SELECT DNO, MONTH, REVENUE,
SUM(REVENUE) OVER (PARTITION BY(DNO)) AS TOTAL_REVENUE
FROM FINANCE;
Window function over all departments—keep all the rows in output
SELECT DNO, MONTH, REVENUE,
SUM(REVENUE) OVER () AS TOTAL_REVENUE
FROM FINANCE;
